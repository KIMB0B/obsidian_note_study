# 특징

> MongoDB에서는 데이터가 Document로 불리며, 이 데이터의 집합을 Collection (RDMS에서는 Table)이라고 함
> 
> RDBMS에서 지원하지 않던 형태로도 저장할 수 있기 때문에 관계를 연결하는 JOIN이 필요 없이 한 문서에 좀 더 이해하기 쉬운 형태 그대로 정보를 저장할 수 있음
> 
> 객체지향 프로그래밍과 잘 맞고 JSON을 사용할 때 아주 유용함
